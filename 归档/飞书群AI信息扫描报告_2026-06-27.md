### 5. 缓存命中率优化：Append-Only Loop + 少改 System Prompt

**原文关键句：**
- 曲水流觞TechRill 文章："Hermes 越用越快的原因：System Prompt 和固定上下文通过 KV 缓存复用，每次对话少算 Prefill。"
- 曲水流觞TechRill 文章："System Prompt 首 token 改一个字，整段缓存全部失效，成本暴涨。"
- 聪哥："缓存命中率 99，挺高的。"

**总结点评：** DeepSeek 等 API 的 Prefix Cache 按字节前缀匹配。如果 Agent 框架每轮都压缩历史、重排提示词、插入时间戳，缓存会失效。Reasonix 的"Append-Only Loop"把历史消息只追加不修改，实测 cache hit ratio 达 99.82%。

**【对二哥的价值】** ⭐⭐⭐ 二哥如果跑长报告/批量文案生成，保持 System Prompt 和前置上下文稳定，能显著降低 token 费。如果 WorkBuddy 有缓存机制，尽量不要频繁修改身份文件或系统提示。

## 五、总结与可执行建议

### 对二哥最相关的 5 条

1. **立即试用海螺 Hailuo 2.3 / MiniMax M3**：视频生成能力可能直接服务短视频/影视剪辑业务。建议让 Agent 生成几个样片对比现有渲染链。
2. **给 WorkBuddy 写/优化 SOUL.md**：参考 170 行模板，明确身份、语气、反驳规则、自主边界、任务地图，让 Agent 从助手变队友。
3. **建立内容生产 Profile 流水线**："选题雷达 → 研究 → 脚本作者 → 配图 → 审校"五个分身，各自绑定不同模型，降低成本并提升质量。
4. **定期清理 Skill**：每月归档未使用 Skill，创建新 Skill 前必须确认，避免上下文膨胀导致 Agent 变蠢。
5. **配置 Agent Mail 作为信息投递口**：用 `agent.qq.com` 给 WorkBuddy 一个专属邮箱，自动接收、分类、归档外部素材和邮件摘要。

- **DeepSeek V4 Flash 安全词升级**：sensenova 平台容易触发 400，建议准备 opencode/阶跃星辰备用。
- **OpenClaw 新版僵尸 Bug**：网关可能假死，关键任务不要单点依赖。
- **本地模型工作流不稳定**：旧手机/本地 PC 部署方案省钱但维护成本高，关键内容生产仍建议用云 API。
